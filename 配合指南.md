## 📋 第一步：配置 Supabase（预计 10 分钟）

### 1.1 创建 Supabase 账号和项目

**操作位置**：浏览器

**步骤**：

1. 打开浏览器，访问：https://supabase.com
2. 点击右上角 **"Start your project"** 或 **"Sign Up"**
3. 使用 GitHub 或 Email 注册/登录
4. 登录后，点击 **"New Project"**
5. 填写项目信息：
   - **Name**: `ai-headhunter`（可自定义）
   - **Database Password**: 点击 "Generate a strong password" 生成强密码，**务必保存好**
   - **Region**: 选择 `Northeast Asia (Tokyo)` 或 `Singapore`（离中国最近）
   - **Pricing Plan**: 选择 **"Free"**（免费套餐够用）
6. 点击 **"Create new project"**
7. **等待项目创建完成**（约 1-2 分钟，显示 "Preparing your project..."）

**预期结果**：项目创建完成后，自动进入 Project Dashboard。

---

### 1.2 获取 Supabase 配置信息

**操作位置**：Supabase Dashboard（浏览器）

**步骤**：

1. 在 Project Dashboard，点击左侧菜单最下方的 **"Settings"（齿轮图标）**
2. 在 Settings 页面，点击 **"API"**
3. 你会看到两个重要的配置区块：

   #### 区块 A：`Project URL` 和 `anon` / `public` key
   - **URL**: `https://xxxxx.supabase.co`（复制这个值）
   - **anon/public**: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`（复制这个值）

   #### 区块 B：`service_role` key（⚠️ 保密，不要泄露）
   - **service_role**: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`（复制这个值）

4. **打开项目根目录下的 `.env.local` 文件**（用 VS Code 或任何文本编辑器）
5. 修改以下三行，**替换为真实值**：

```env
# 修改前（占位符）
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

# 修改后（真实值示例）
NEXT_PUBLIC_SUPABASE_URL=https://abcdefg.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzd...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzd...
```

6. **保存文件**（`Ctrl+S`）

**预期结果**：`.env.local` 文件中的三个 Supabase 变量已填写真实值。

## 📋 第二步：配置 OpenAI API Key（预计 5 分钟）

### 2.1 获取 OpenAI API Key

**操作位置**：浏览器

**前提条件**：你已有 OpenAI 账号和已绑定付款方式（信用卡）。

**步骤**：

1. 打开浏览器，访问：https://platform.openai.com/api-keys
2. 登录 OpenAI 账号
3. 点击 **"Create new secret key"**
4. 在弹出框中：
   - **Name**: 填写 `ai-headhunter`（方便识别）
   - **Permissions**: 保持 `All`（默认）
5. 点击 **"Create secret key"**
6. **立即复制生成的 API Key**（`sk-proj-xxxxx` 开头）
   > ⚠️ 这个 Key 只显示一次，务必保存好！如果丢失，只能重新创建。

7. **打开项目根目录下的 `.env.local` 文件**
8. 修改以下行：

```env
# 修改前
OPENAI_API_KEY=sk-your-openai-api-key

# 修改后（真实值示例）
OPENAI_API_KEY=sk-proj-abcdefghijklmnopqrstuvwxyz
```

9. **保存文件**

**验证 OpenAI 账户余额**：
- 访问：https://platform.openai.com/account/billing/overview
- 确认有至少 `$5` 余额（gpt-4o 约 $0.005/1K tokens）

**预期结果**：`.env.local` 文件中的 `OPENAI_API_KEY` 已填写真实值。

## 📋 第三步：初始化数据库（预计 5 分钟）

### 3.1 执行数据库 Schema

**操作位置**：Supabase Dashboard > SQL Editor（浏览器）

**步骤**：

1. 打开浏览器，访问：https://supabase.com/dashboard（如果已退出登录，请重新登录）
2. 点击进入你的项目
3. 在左侧菜单，点击 **"SQL Editor"**（图标是一个播放按钮 ▶）
4. 点击 **"New Query"**（新建查询）
5. **打开项目文件夹中的 `supabase/schema.sql` 文件**（用 VS Code 或任何文本编辑器）
6. **全选并复制** 文件中的所有内容（`Ctrl+A` → `Ctrl+C`）
7. 回到 Supabase SQL Editor，将内容粘贴到查询编辑器中（`Ctrl+V`）
8. 确认粘贴的内容完整后，点击右上角 **"Run"** 按钮
9. **等待执行完成**（约 10-30 秒）

**预期结果**：
- 编辑器下方显示 **"Success. No rows returned."** 或类似成功消息
- 没有红色错误信息

## 📋 常见问题排查

### Q1: 启动开发服务器时提示 "Port 3000 is in use"

**解决方法**：

```bash
# 方法 1：使用其他端口
npm run dev -- -p 3456

# 方法 2：找到并停止占用 3000 端口的进程
# Windows (PowerShell)
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# Mac/Linux
lsof -ti:3000 | xargs kill -9
```

---

### Q2: Supabase 连接失败（页面显示 "AuthApiError"）

**原因**：`.env.local` 中的 Supabase URL 或 Key 不正确。

**解决方法**：
1. 重新复制 Supabase Dashboard > Settings > API 中的值
2. 确保 `.env.local` 中没有多余的空格或引号
3. 重启开发服务器

---

### Q3: 简历上传失败（"Bucket not found"）

**原因**：Storage Bucket 未创建。

**解决方法**：参考第四步，在 Supabase Dashboard > Storage 中手动创建 `resumes` 和 `jd-files` Bucket。

---

### Q4: OpenAI API 调用失败（"Insufficient quota"）

**原因**：OpenAI 账户余额不足。

**解决方法**：
1. 访问 https://platform.openai.com/account/billing/overview
2. 点击 "Add payment method" 或 "Add credits"
3. 充值后重试
